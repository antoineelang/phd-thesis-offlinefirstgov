# Blockchain Architecture for Securing E-Government Services in Cameroon

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

This repository serves as the central Technical Architecture Dossier for my doctoral thesis. The research focuses strictly on designing hierarchical, offline-first blockchain architectures capable of securing critical e-government services (e.g., civil registries, land tenure) in severely constrained environments characterized by frequent power and telecommunication outages.

## 🧭 Repository Navigation

To maintain clear separation of concerns, the artifacts, simulations, and hardware testbed logs for each specific publication are isolated in their own branches. **Please select the corresponding branch from the GitHub dropdown menu to access the code and specific documentation for each article.**

### 🌿 Branches & Publications

* **[Branch: `article1-tetralemma`]** *Article 1: Limitations of Layer 2 Blockchain Solutions in Constrained Environments: Bridging the Physical Efficiency Gap* (Submitted, 2026).
    Contains Monte Carlo simulations modeling security degradation, sensitivity analysis, and the ZK-Rollup Energy Wall.
* **[Branch: `article2-dag-dsr`]** *Article 2: A Hierarchical Offline-First Blockchain Architecture for E-Government Services: Mitigating Intermittent Connectivity in Cameroon* (Submitted, 2026).
    Contains the Directed Acyclic Graph (DAG) Deterministic State Reconciliation (DSR) simulator and the 52 KB asynchronous payload generator.
* **[Branch: `article3-edge-drl`]** *Article 3: Resource-Aware Blockchain Orchestration: Edge-Level Machine Learning for Constrained E-Government Infrastructure* (Submitted, 2026).
    Contains the SimPy discrete-event battery simulation, the INT8-quantized orchestrator, the Rust L2 Sequencer scaffold, and raw INA219 hardware metrology logs.
